# Comprehensive CRDT Strategies Catalog

Here is a highly expanded, categorized list of CRDT (Conflict-free Replicated Data Type) strategies. This serves as a massive (loose) roadmap for making `Ama.CRDT` the most awesome and feature-complete distributed data structures library.

Strategies already implemented in `Ama.CRDT` are marked with `[x]`. They are broken down into core base strategies and flexible decorators.

I am weighting on what to expand the library with next, feel free to open an issue if you feel that there is something you want prioritized.

## Base Strategies

### 🗃️ Registers (Single Values)
- [x] **LWW Register** (Last-Writer-Wins)
- [x] **FWW Register** (First-Writer-Wins)
- [x] **Max-Wins Register** (Resolves to the highest value)
- [x] **Min-Wins Register** (Resolves to the lowest value)
- [x] **Average Register** (Maintains an average based on node contributions)
- [ ] **MV Register** (Multi-Value Register; keeps concurrent conflicting values until explicitly resolved)
- [ ] **Multi-Author Register** (A variant of MV Register that explicitly attributes values to their origin author)
- [ ] **Enable-Wins Flag** (Boolean flag where `true` overrides `false`)
- [ ] **Disable-Wins Flag** (Boolean flag where `false` overrides `true`)
- [ ] **Tied-Resolution Register** (Uses a custom deterministic tie-breaker function)
- [ ] **Versioned Register** (Explicitly bumped integer versions to force wins)
- [ ] **Lexicographic Register** (Resolves conflicts based on alphabetical/lexicographical sorting)
- [ ] **Blind Register** (Unconditionally overwrites with the latest received payload; relies entirely on network order)

### 🔢 Counters
- [x] **G-Counter** (Grow-Only Counter)
- [x] **PN-Counter** (Positive-Negative Counter / `CounterStrategy`)
- [x] **Bounded Counter** (Clamped between Min and Max limits)
- [x] **Vote Counter** (Aggregates unique actor votes)
- [ ] **Fractional Counter** (Optimized for floating-point increments)
- [ ] **Non-Negative Counter** (A PN-counter that naturally bottoms out at 0 instead of going negative)
- [ ] **V-Counter** (Vector Counter; explicitly exposes individual node contributions to the consumer)
- [ ] **EWMA Counter** (Exponentially Weighted Moving Average; tracks velocity/metrics over time)
- [ ] **Lexicographic Counter** (Like fractional indexing, infinitely subdivides space between values)
- [ ] **Modulo / Wrapping Counter** (A counter that guarantees bounds by wrapping around after hitting a maximum, useful for circular references)
- [x] **Reset-Epoch Counter** (A PN-Counter that can be safely reset to 0 across the cluster using causal epochs)
- [ ] **High-Watermark Counter** (Automatically tracks and retains the absolute highest value the counter has ever reached)
- [ ] **Velocity / Rate Counter** (Specifically tracks the rate of increment operations over a defined sliding time window)

## Decorator Strategies (Composable Wrappers)
Decorators wrap existing base strategies to append specific metadata, validation rules, or deterministic side-effects without inherently altering the core strategy logic. Because they are composable, they apply universally to Registers, Collections, or Trees.

#### 🛡️ Validation & Constraints
- [x] **Approval Quorum** (Requires n-node agreement or explicit approvals before applying a change to the underlying CRDT)
- [x] **Epoch Bound** (Restricts operations to a specific sliding time window or generation)
- [ ] **Conditional / Predicate Wrapper** (Only accepts a new operation if a provided predicate function evaluates to true)
- [ ] **Compare-and-Swap (CAS) Wrapper** (Only accepts an operation if the previous state matches an expected causal state/value)
- [ ] **Causal Barrier** (Strictly requires causal context/vector clocks validation before applying operations, dropping or queuing out-of-order data)
- [ ] **Schema Validation** (Rejects operations locally and network-wide if they violate a strict schema definition)

#### ⏳ Lifecycle & Mutations
- [ ] **Sticky / Write-Once Wrapper** (Once the underlying property is set to a non-null value, it blocks all future mutations or deletions)
- [x] **Resettable Wrapper** (Allows clearing an entire underlying CRDT securely via global tombstones or epochs without breaking commutativity)
- [ ] **Reversible / Undoable Wrapper** (Wraps operations in compensatable actions to support distributed Undo/Redo stacks)
- [ ] **Capacity Eviction / LRU** (Deterministically evicts items based on LRU or FIFO when capacity limits of the underlying collection are reached)
- [ ] **Time-To-Live (TTL) Expiration** (Automatically nullifies or deletes the underlying value/keys after a specified time threshold based on timestamp metadata)
- [ ] **Time-Decay Wrapper** (Mathematically decays the underlying numeric value over time based on physical clocks)
- [ ] **Default-Value Fallback** (Provides a fallback or default value when the underlying keys or registers resolve to null)
- [ ] **Tombstone-Aware / Soft-Delete Wrapper** (Explicitly exposes 'deleted' states/tombstones to the consumer instead of hiding them from the data model)

#### 🚰 Data Flow & Processing
- [ ] **Throttling / Debounce** (Collapses high-frequency local operations before broadcasting to the network)
- [ ] **Deduplication Wrapper** (Drops explicitly duplicate intents generated by jittery clients before they bloat the metadata)
- [ ] **Type Coercion / Formatting** (Forces incoming string/numeric operations to adhere to strict formatting rules before application)
- [ ] **Compression** (Transparently compresses large states or patches using algorithms like Zstd or Brotli before writing to metadata)
- [x] **Write-Ahead Log (WAL) Wrapper** (Stages operations durably to disk inside the decorator before allowing the applicator to proceed)

#### 🔒 Security & Privacy
- [ ] **RBAC / Permissions** (Role-Based Access Control; ignores operations from unauthorized replicas dynamically)
- [ ] **Encrypted Payload** (Transparently encrypts the underlying state/operations for zero-knowledge synchronization)
- [ ] **Cryptographic Signature** (Appends and validates asymmetric key signatures for every operation routed to the underlying CRDT)
- [ ] **Anonymizer / Obfuscation** (Deterministically hashes or masks PII fields inside the payload before committing them to the state)

#### 🔍 Observability & History
- [x] **Audit Log** (Creates a secondary immutable history stream of all operations applied to the target)
- [ ] **Historical / Time-Travel Wrapper** (Retains a bounded timeline log of past states/values applied, allowing point-in-time queries)
- [ ] **Observable / Event Emitter Wrapper** (Emits explicit events only when the resolved value changes, masking underlying concurrent updates that don't change the outcome)
- [ ] **Threshold Observer** (Emits localized deterministic events once the underlying CRDT reaches a specified value or state)
- [ ] **Projection** (Read-only transformation of the underlying state for specialized queries)

## 📊 Domain-Specific & Probabilistic

- [ ] **HyperLogLog CRDT** (Probabilistic cardinality estimation; *Composed by nesting Max-Wins Register properties within a Fixed Size Array collection.*)
- [ ] **Bloom Filter CRDT** (Probabilistic membership testing; *Composed by nesting boolean flags mapped with Max-Wins Register inside a Fixed Size Array.*)
- [ ] **Count-Min Sketch CRDT** (Probabilistic frequency tracker; *Composed by nesting G-Counter values within a Fixed Size Array collection.*)
- [ ] **Top-K CRDT** (Probabilistic leaderboard/top items; *Composed by wrapping a Sorted Set or combining a Max-Wins Map.*)
- [ ] **Rate Limiter / Token Bucket CRDT** (Distributed API rate limiting; *Achieved seamlessly by stacking an Epoch Bound decorator directly over a PN-Counter.*)
- [ ] **Auction / Bidding CRDT** (Strictly monotonic highest-bidder tracker; *Built by applying a Max-Wins Register combined with custom logic or an Approval Quorum decorator for settlement.*)
- [ ] **Shopping Cart CRDT** (Domain-optimized map handling item merges, max quantities, and checkouts; *An OR-Map containing nested POCOs with Bounded Counters for quantities and LWW Registers for item details.*)
- [ ] **Inventory / Stock CRDT** (Constraint-based counter; *A POCO combining a Bounded Counter for totals and a 2P-Set for tracking fulfilled reservation IDs.*)
- [ ] **Calendar / Booking CRDT** (Time slots, duration tracking, and priority-based conflict resolution; *An OR-Map mapping POCO intervals with LWW Register start/end times.*)
- [ ] **Playlist CRDT** (Queue + reordering + current track pointer; *A POCO mixing an LSEQ (or Array LCS) for ordered tracks and a shared LWW Register pointer to the active index.*)
- [ ] **Matrix CRDT** (2D array optimized for Spreadsheet collaboration and formula graphs; *Composed easily via a Fixed Size Array containing another Fixed Size Array.*)
- [ ] **Time-Series CRDT** (Append-only timestamped analytics events; *Constructed using a G-Set holding immutable event POCOs.*)

- [ ] **IoT Device Twin CRDT** (Manages 'desired' vs 'reported' hardware states; *A POCO with two nested maps (Desired/Reported), using State Machine CRDTs for transition rules.*)
- [ ] **Role-Based ACL CRDT** (Distributed Access Control List; *An OR-Map where keys are UserIDs and values are 2P-Sets of granted permission enums.*)
- [ ] **Survey / Polling CRDT** (Dynamic surveys with aggregated multi-choice answers; *An OR-Map of Question POCOs containing nested Vote Counters or Counter-Maps for the answers.*)
- [ ] **Metrics / Histogram CRDT** (Distributed latency and measurement bucketing; *An OR-Map where dictionary values are nested G-Counter frequencies.*)
- [ ] **T-Digest CRDT** (Probabilistic structure for computing highly accurate quantiles/percentiles in distributed systems; *An OR-Map of weighted centroid POCOs aggregated via Average Registers.*)
- [ ] **MinHash CRDT** (Probabilistic structure for estimating the Jaccard similarity / overlap between distributed datasets; *Composed via a Fixed Size Array holding Min-Wins Register properties.*)
